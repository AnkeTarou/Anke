/vote/

  • 指定した投稿(アンケート)に投票を行う

形式

POST

受信オブジェクト

{
  id:"投稿ID",
    index:[0],
  user:{session:"セッションキー",_id:"ユーザーID"}
}

送信オブジェクト

{
  id:"投稿ID",
  answers:[                         //選択肢の配列
     {answer:"選択肢",total:1},            //選択肢と投票された総数
     {answer:"選択肢",total:1},
     {answer:"選択肢",total:1}
  ],
  comment:[
    {
      senderId:"コメント送信者ID",
            content:"内容",
            commentId:"コメントID"
    }...
  ],
  good:2,                        //いいなされた数
  total:3,                            //総投票数
}

動き

  1. ブラウザからリクエスト受け取る
  2. userの認証が完了
  3. 投稿IDが存在する
  4. userはその投稿IDに対してまだ投票したことがない
  5. typeにそったindexになっている
  6. indexが正しい値を指している
  7. Data.questionの投稿IDのanswersのindexまたは[index,index...]番目のvoterにuserの_Idを追加する
  8. 送信オブジェクトを作成しブラウザにレスポンスを行う

エラー

{state:"error",error:"エラーが起きた番号"}をブラウザに送信する

/post/

  • アンケートの投稿を行う。
  • ユーザー認証あり

形式

POST

受信オブジェクト

{
  query:"アンケート内容"
  answers:["選択肢1","選択肢2","選択肢3"....],
  answerType:"radio" or "checkbox",
  user:{session:"セッションキー",_id:"ユーザーID"}
}

送信オブジェクト

{
    state:"success" or "error"
}

動き

  1. ブラウザからリクエスト受け取る
  2. userの認証が完了
  3. queryの長さが255以下1以上
  4. queryの値に特殊文字が含まれていれば置換する
  5. answersが配列であるかの確認
  6. answersの配列の長さが48以下1以上
  7. answersの配列の中の値の長さが127以下1以上の文字列
  8. answerTypeの値が"radio","checkbox"のどちらか
  9. answersの配列の中の値に特殊文字が含まれていれば置換する
  10. Data.questionの形式に成型しData.questionに追加する
  11. 追加に成功したら{state:"success"}をブラウザに送信する

エラー

{state:"error",error:"エラーが起きた番号"}をブラウザに送信する

/login/

  • ログインを行う

形式

POST

受信オブジェクト1

{
  _id:"ユーザーID",            //入力されたユーザーID
  pass:"パスワード",            //入力されたパスワード
  type:"login",                //ログイン形式
}

受信オブジェクト2

{
  session:"セッションキー",
  user:"ユーザーID",            プロパティ名変更！！！！！
  type:"session"            //ログイン形式
}

送信オブジェクト

{
  state:"success" or "error,
  session:"新しいセッションキー",
}

動き

  1. ブラウザからリクエスト受け取る
  2. typeが"login"なら3 ,"session"なら5へ
  3. Data.userを_idとpassで検索する
  4. 見つかれば{state:"success",session:"新しいセッション"}をブラウザに送信する
  5. Data.userをsessionで検索し見つかった値の_idとuserが一致する
  6. {state:"success",session:"新しいセッションキー"}をブラウザに送信する

エラー

{state:"error",error:"エラーが起きた番号"}をブラウザに送信する

/account/

  • ユーザー登録を行う

形式

POST

受信オブジェクト

{
  _id:"ユーザーID",              //入力されたユーザーID
  pass:"パスワード"            //入力されたパスワード
  gender:1 or 2,        //性別    １= 男, ２= 女
  age:20,
  area:"地域",
  birthday:"誕生日"
}

送信オブジェクト

{
    state:"success" or "error"
}

動き

  1. ブラウザからリクエスト受け取る
  2. _idでData.user内を検索し、_idの重複がないことを確認する
  3. passが空白ではない
  4. genderが1か2である
  5. ageが正しい値を指している
  6. areaが正しい値を指している
  7. birthdayが正しい値を指している
  8. Data.userに登録する
  9. {state:"success"}をブラウザに送信する

エラー

{state:"error",error:"エラーが起きた番号"}をブラウザに送信する

/search/

  • アンケートを検索する
  • 初期は投票数の降順
  • ユーザー認証できれば投稿したことがあるものは結果も帰ってくる

形式

POST

受信オブジェクト

{
  value:"検索文字列",
  sort:"total" or "good" or "date",        //投票数 good数　日時
  order:1 or -1,                      //1 = asc , -1 = desk
  user:{session:"セッションキー",_id:"ユーザーID"}
}

送信オブジェクト

[
  {
    result:false,
    _id:"投稿ID",
    query:"質問内容",
    type:"投稿形式",            //radio or checkbox
    answers:[
       {answer:"選択肢１"},
       {answer:"選択肢２"},
       {answer:"選択肢３"}
    ],
    total:6,                //総投票数
    good:3,                //いいな数
    date:ISODate("2018-06-12T00:53:55.393Z")
  },
  {
    result:true,
    id:"投稿ID",
    query:"質問内容",
    type:"投稿形式"                          //radio or checkbox
    answers:[
      {answer:"選択肢",total:1},            //選択肢と投票された総数
      {answer:"選択肢",total:1}
      {answer:"選択肢",total:1}
    ],
    comment:[
      {
        senderId:"コメント送信者ID",
        content:"内容",
        commentId:"コメントID"
      }...
    ],
    good:2,
    total:3,
    date:ISODate("2018-06-12T00:53:55.393Z")
  }...
]

動き

  1. ブラウザからリクエスト受け取る
  2. userの認証を行う。userの中身がなければ行わない
  3. sortが"total","good","date"のいずれかであるかを確認する
  4. orderが1,-1かを確認する
  5. valueを文字列に変更する
  6. valueでData.question内を検索し、sortのorder順の上から15件を取得する
  7. ユーザー認証していれば検索結果の中で投票したことがある内容はresultをtrueにし、commnetを追加し、answersにtotalを追加する
  8. 送信オブジェクトを返す

エラー

{state:"error",error:"エラーが起きた番号"}をブラウザに送信する

/update/

  • グラフを更新する
  • 性別・年齢・地域別データで返すことも可能
  • ユーザー認証あり

形式

POST

受信オブジェクト

{
  user:{session:"セッションキー",_id:"ユーザーID"},
  update:[
  //format    1 = nomal,2 = gender,3 = age,4 = area
    {_id:"更新したい投稿ID",format:"1"},
    {_id:"更新したい投稿ID",format:"2"},
    {_id:"更新したい投稿ID",format:"3"}...
  ],
}

送信オブジェクト

[
  {
    _id:"投稿ID",
    label:[],
    date:[                         //選択肢の配列
      {value:[],total:40},
      {value:[],total:30},
      {value:[],total:20}
    ],
    total:90                    //投票した人数
  },
  {
    _id:"投稿ID",
    label:["男性","女性"],
    date:[                         //選択肢の配列
      {value:[25,15],total:40},
      {value:[10,20],total:30},
      {value:[5,15],total:20}
    ],
    total:90                    //投票した人数
  },
  {
    _id:"投稿ID",
    label:["10代","20代","30代"...],
    date:[                         //選択肢の配列
      {value:[22,1,14...],total:40},
      {value:[3,1,5...],total:30},
      {value:[5,1,3...],total:20}
    ],
    total:120                    //投票した人数
  }
]

動き

  1. ブラウザからリクエスト受け取る
  2. userの認証を行う
  3. updateが配列であることを確認する
  4. updateの中身のobjectに_idとformatがあることを確認する
  5. updateの配列の全ての要素に下記を行う
    1. _idとformatが存在するかどうかを確認する
    2. _idがData.question内にあることを確認する
    3. _idにすでに投票したことがあるかを確認する
    4. _idの投稿を取得してformatに送信オブジェクト作成に必要なデータを収集する
    5. 送信オブジェクトに追加する
  6. 送信オブジェクトをブラウザに返す

エラー

{state:"error",error:"エラーが起きた番号"}をブラウザに送信する

/follow/

  • followする

形式

POST

受信オブジェクト

{
  user:{session:"セッションキー",_id:"ユーザーID"},
  targetId:"対象ユーザーID",
  follow:true or false      //する = true , やめる = false
}

送信オブジェクト

{
    state:"success" or "error"
}

動き

  1. ブラウザからリクエスト受け取る
  2. userの認証を行う
  3. targetIdがData.userの_idに存在するかを確認する
  4. followがtrueなら1へ、falseなら2へ
      • Data.user内の_id = user._idのfollowの配列にtargetIdを追加する
      • Data.user内の_id = targetId._idのfollowerの配列にuser._idを追加する
      • Data.user内の_id = user._idのfollowの配列の中のtargetIdを削除する(空白は詰める)
      • Data.user内の_id = targetId._idのfollowerの配列のuser._idを削除する(空白は詰める)
  5. {state:"success"}をブラウザに返す

エラー

{state:"error",error:"エラーが起きた番号"}をブラウザに送信する
